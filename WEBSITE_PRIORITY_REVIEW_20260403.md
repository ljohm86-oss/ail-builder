# Website Priority Review 2026-04-03

## Purpose

This document re-ranks the supported website lines and next investment options after the managed / unmanaged + hook workflow was promoted to a stable milestone on 2026-04-03.

It exists to answer:

- which website line is strongest in authored polish
- which line is strongest in mainstream homepage quality
- which line is strongest in structural/runtime proof
- which line is strongest as a bounded operational surface
- which next investment now has the highest leverage

## Short Answer

The ranking itself did not change much.
What changed is the leverage of the non-website-line work.

- Personal Independent Site is still the strongest authored-polish line.
- Company / Product Website is still the strongest mainstream homepage line.
- Ecommerce Independent Storefront is still the strongest structural/runtime proof line.
- After-Sales Service Website is still the strongest bounded tracked-case surface.
- Managed/unmanaged + hooks are now the strongest productization layer around the website frontier.

That means the next decision is even less about rescuing a website line and even more about choosing the next phase deliberately.

## Current Ranking

### 1. Personal Independent Site

Current status:

- strongest authored-polish line

Best use:

- best proof of page craft, authorship, curation, and first-impression quality

### 2. Company / Product Website

Current status:

- strongest mainstream homepage line

Best use:

- best proof of high-quality product-marketing homepage structure
- best proof of staged proof / FAQ / intake / closure flow
- best line if we want to keep pushing brand posture and homepage distinction

### 3. Ecommerce Independent Storefront

Current status:

- strongest structural and runtime continuity line

Best use:

- strongest evidence that AIL supports continuous multi-page website behavior
- strongest proof of discovery-to-purchase continuity inside the supported surface

### 4. After-Sales Service Website

Current status:

- strongest bounded tracked-case / operational-surface line

Best use:

- proof of bounded service-center and tracked-case support without claiming full app/platform behavior

## Highest-Leverage Next Investment

If the goal is the highest-leverage next product investment, the best current choice is still:

- durable customization productization around managed/unmanaged + hooks

But the nuance is different now.

On 2026-04-02 this was the strongest next direction.
On 2026-04-03 it is now also a valid pause point.

Why:

- website quality across all four lines is already strong enough to act as a credible milestone frontier
- the durable customization path is now strong enough to be treated as a real product capability
- the next move does not need to be emergency productization rescue
- the next move can now be a more deliberate phase-opening choice

## Best Website-Line Investment If We Stay On Website Quality

If we explicitly stay on website quality rather than customization/productization, the best next line to invest in is still:

- Company / Product Website

Why:

- it is the broadest mainstream proof line
- it still has the most upside in stronger authored-brand distinction without needing new route families
- stronger company-brand distinction would improve both demo quality and market clarity

## Best Productization Follow-Up If We Stay On Customization

If we explicitly stay on customization/productization rather than website aesthetics, the best next line of work would be:

- more guided hook selection and scaffolding UX

Examples:

- selecting one suggestion interactively and immediately scaffolding it
- better workspace-level filtering and operator summaries
- selectively expanding runtime context only for the highest-value hooks

Why:

- the current workflow is already viable
- the next biggest gains are now about reducing friction, not inventing the boundary itself

## Best Proof-Consolidation Work

If the goal is not new capability but cleaner phase packaging, the best next work is:

- one final packaging / handoff / documentation normalization pass

Why:

- the core quality frontier is already stronger than many older review docs imply
- the productization frontier is also stronger now
- the repo would benefit from one cleaner operator-facing "current truth" layer that unifies both of those realities

## Recommended Focus Order

The best current order is:

1. package the 2026-04-03 frontier cleanly
2. decide whether next phase means:
   - stronger brand distinction
   - deeper customization ergonomics
   - or broader platform capability
3. if staying on website quality, keep Company / Product Website as the main distinction push
4. keep Ecommerce as the strongest continuity/runtime proof line
5. keep Personal as the strongest authored-polish proof line
6. keep After-Sales as the strongest bounded tracked-case proof line

## One-Line Conclusion

As of 2026-04-03, Personal is still the strongest authored-polish line, Company is still the strongest mainstream homepage line, Ecommerce is still the strongest continuity/runtime line, After-Sales is still the strongest bounded tracked-case line, and the most important shift is that managed/unmanaged + hooks now count as a stable milestone rather than only the next thing that needs to be fixed.
