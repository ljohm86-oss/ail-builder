# AIL Project Structure

## Overview

This document explains the standard directory layout of an AIL-managed project and the ownership boundary between compiler-managed files and user-managed files.

## Standard Layout

```text
project/
 ├─ src/
 │   ├─ views/
 │   │   └─ generated/
 │   ├─ router/
 │   │   └─ generated/
 │   ├─ custom/
 │   ├─ extensions/
 │   └─ theme/
 ├─ backend/
 │   ├─ generated/
 │   └─ custom/
 ├─ .ail/
 │   ├─ source.ail
 │   ├─ manifest.json
 │   ├─ last_build.json
 │   └─ patches/
```

## Managed Zones

Managed zones are controlled by AIL compile output.

```text
src/views/generated/
src/router/generated/
backend/generated/
src/ail_generated/
```

### Managed Zone Rules

| Rule | Meaning |
| --- | --- |
| Rebuildable | Files can be regenerated from AIL |
| Overwriteable | Cloud sync may replace them |
| Deletable | Missing files from latest manifest may be removed |
| Manifest-tracked | Ownership and hashes must be recorded |

## Custom Zones

Custom zones are reserved for user-authored logic and styling.

```text
src/custom/
src/extensions/
src/theme/
backend/custom/
```

### Custom Zone Rules

| Rule | Meaning |
| --- | --- |
| User-owned | Compiler must not overwrite |
| Stable | Sync should ignore these files |
| Extension-friendly | Can wrap or compose generated output |
| Free-edit | Users may refactor without cloud restrictions |

## .ail Project Directory

The `.ail/` directory stores project coordination metadata.

### Recommended Files

| File | Purpose |
| --- | --- |
| `.ail/source.ail` | Canonical AIL source |
| `.ail/manifest.json` | Managed file ownership and hashes |
| `.ail/last_build.json` | Local record of last accepted build |
| `.ail/patches/` | Optional patch records or scoped compile input |

## IDE Recognition Rules

The IDE should identify generated files using both path and manifest.

### Path-based Detection

Files under these roots should be treated as generated by default:

- `src/views/generated/`
- `src/router/generated/`
- `backend/generated/`
- `src/ail_generated/`

### Manifest-based Detection

The IDE should also check `.ail/manifest.json` to confirm:

- whether a file is managed
- which build last wrote it
- which checksum is expected

## Recommended IDE Behavior

| File Type | IDE Behavior |
| --- | --- |
| Managed file | Show generated badge, warn on edit, allow conflict-aware sync |
| Custom file | Treat as normal user code |
| Manifest file | Treat as internal metadata |
| `.ail/source.ail` | Treat as canonical project source |

## Practical Rule

Generated code should be edited only by changing AIL or by extending generated output from custom zones.
