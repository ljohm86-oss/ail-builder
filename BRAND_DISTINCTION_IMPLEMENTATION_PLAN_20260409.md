## Purpose

This document turns the current `Brand Distinction Phase` recommendation into a concrete first-cut implementation plan.

It exists to answer:

- where to start
- what to change first
- how to measure success without drifting into vague aesthetics work
- what should count as done for the first pass

## Short Answer

The safest first cut is:

1. start on the company / product line first
2. improve distinction through clearer brand language, stronger page posture, and more recognisable visual identity
3. then use the personal line as the second validation track for authored distinction

The first pass should stay inside the current supported website scope.
It should not open broader platform work, new route families, or complex new runtime behavior.

## Core Phase Question

Can AIL move from:

- strong, believable websites

to:

- more recognisably authored and branded websites

without weakening:

- structure
- runtime continuity
- current supported scope

## Phase Scope

### In Scope

- stronger brand posture on company / product sites
- stronger authored signature on personal sites
- more visible visual systems where they improve identity
- better variation in tone and page posture between the strongest website baselines
- prompt / generator / presentation changes that stay inside the current website surface

### Out Of Scope

- new supported route families
- broader platform or app-like scope
- backend or operations-system expansion
- large customization workflow redesigns
- generic style churn without measurable distinction gain

## Phase Success Criteria

This phase should count as successful only if all of the following become true.

### 1. Stronger First-Glance Differentiation

At first glance:

- the company line feels less like a strong but safe homepage system
- the personal line feels less like a refined template and more like a authored surface
- the strongest baselines become easier to distinguish from one another visually and rhetorically

### 2. Stronger Point Of View

The output should show more obvious opinion and posture through:

- headline framing
- section naming
- proof framing
- CTA language
- closing language

without becoming noisy or unstable.

### 3. Visual Identity Should Carry More Meaning

Improvement should come through more than one accent line.
The strongest pages should show clearer identity via:

- typographic hierarchy
- color decisions
- rhythm and spacing posture
- recurring visual motifs
- stronger section transitions where justified

### 4. Mainstream Stability Must Hold

The phase fails if distinction gains come from losing the qualities that currently make the baselines believable:

- clear structure
- readable hierarchy
- sensible proof flow
- usable runtime paths
- mobile viability

1. the company line is clearly more distinctive without losing mainstream credibility
2. the work starts drifting into novelty for novelty's sake
3. improvements stop being visible without side-by-side explanation
4. new changes begin threatening structure or runtime stability
